java面试-String StringBuffer与StringBuilder之间区别

2018-10-05   面试 

运行速度

StringBuilder > StringBuffer > String

原因

String为不可变字符序列，而StringBuilder和StringBuffer均为可变字符序列，即String对象一旦创建之后该对象是不可更改的，但后两者的对象是变量，是可以更改的。

代码分析

public class Demo {

    public static void main(String[] args) {

        String str="木子李";

        StringBuilder sb1=new StringBuilder("木子李");

        StringBuffer sb2=new StringBuffer("木子李");

        System.out.println(test(str));

        System.out.println(test(sb1));

        System.out.println(test(sb2));

    }

    //测试效率方法，追加1000次字符测时间
    static long test(CharSequence charSequence) {
        long start=System.currentTimeMillis();
        if(charSequence instanceof String) {
            String str=(String) charSequence;
            for(int i=0;i<1000;i++) {
                str+="木子李";
            }
        }else if(charSequence instanceof StringBuilder){
            StringBuilder sb= (StringBuilder) charSequence;
            for(int i=0;i<1000;i++) {
                sb.append("木子李");
            }
        }else if(charSequence instanceof StringBuffer) {
            StringBuffer sb=(StringBuffer) charSequence;
            for(int i=0;i<1000;i++) {
                sb.append("木子李");
            }
        }
        long end=System.currentTimeMillis();
        return end-start;
    }
}

解释

  • String 是字符串常量，它的 + 操作会重新创建对象，追加1000次，一共创建了1001个对象，回收1000个对象，所以效率低
  • StringBuilder和StringBuffer是可变字符序列，内容可以修改，不会创建新对象，追加1000次，一共创建了2个对象
  • String的赋值速度是要远高于StringBuilder和StringBuffer，因为String里面的Char[] value的长度是根据字符的长度直接确定的，而可变序列的char[] value是会根据字符的长度进行扩容
  • StringBuffer线程安全，效率相对要低，StringBuilder线程不安全，效率相对要高
    注意：平常使用String进行字符串拼接时，为了提高系统性能，我们可以采用String.format(“我的名字叫%s”, “木子李”);

线程

  • 在线程安全上，StringBuilder是线程不安全的，而StringBuffer是线程安全的
  • 如果一个StringBuffer对象在字符串缓冲区被多个线程使用时，StringBuffer中很多方法可以带有synchronized关键字，所以可以保证线程是安全的，但StringBuilder的方法则没有该关键字，所以不能保证线程安全，有可能会出现一些错误的操作。所以如果要进行的操作是多线程的，那么就要使用StringBuffer，但是在单线程的情况下，还是建议使用速度比较快的StringBuilder。

总结

String：适用于少量的字符串操作的情况

StringBuilder：适用于单线程下在字符缓冲区进行大量操作的情况

StringBuffer：适用多线程下在字符缓冲区进行大量操作的情况

注：关于String，StringBuffer，StringBuild的char[] value扩容情况详情请阅读相关源码，此处不做赘述。
